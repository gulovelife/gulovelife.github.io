Set、WeakSet、Map、WeakMap

Posted on 2020-08-10

介绍

这里算是阮一峰老师es6入门那个书里的Set和Map那一章的学习笔记，不会记的很全，但是是查漏补缺。

内容

下面就开始吧

Set

  • set可以接受一个数组或者具有iterable接口的其他数据结构作为参数，用来初始化。dom也可以。
  • 可以用[...new Set(array)]来做去重
  • set在add的时候不会发生类型转换，类似于===，但是还是略有不同，NaN和NaN是相同的。两个对象总是不相同的，不管是不是空对象。
  • set.add返回set本身，所以add可以连续调用
  • Array.from可以将set转数组
  • 遍历操作，keys()/values()/entries()/forEach()
  • 并集new Set([...a, ...b])
  • 交集new Set([...a].filter(x => b.has(x)))
  • 差集new Set([...a].filter(x => !b.has(x)))
  • 想要遍历的时候同步改变set结构，只能用原来的set映射出新的结构，然后改变之后再复制给set

WeakSet

  • 和Set类似，但是只能放对象
  • 为什么叫WeakSet呢，是因为WeakSet中的对象都是弱引用，垃圾回收机制不考虑WeakSet中的引用，如果其他的对象不引用该对象，那么垃圾回收机制会自动将其回收。也就是说，只要引用内部对象的外部对象消失，那么内部对象也会消失。由于这种特性，所以WeakSet不能遍历，没有size，forEach等。
  • WeakSet可以用来存储dom节点，因为不用担心节点从文档移出时，会引发内存泄漏。

Map

  • 为什么要有map呢，因为对象只能用字符串作为键，如果用一个对象作为键的话，就会被转化成[object, Object]。这是我们不希望的。所以出现了map，map可以让各种类型的值作为键。
  • const map = new Map([['name': '张三'], ['age': 25]])。如果用new Map()这种方式新建一个map的话，里面的值必须是个键值对的方式，比如['key', 'value']。
  • 对于同一个对象的引用，map才将其视为一个键，如果只是内容相同的对象，map会视为两个键，因为引用的内存地址不一样。map的键跟内存地址绑定。如果不是引用类型，那需要严格相当。对于NaN则视为同一个键。
  • 遍历方法和set一样。map的forEach方法接收第二个参数，绑定this。
  • 对象转为map，可以用Object.entries()。let obj = {b: 1}; Object.entries(obj) // [['a': 1]]
  • JSON和Map的转换，需要注意键是不是字符串

WeakMap

  • 只接受除了null的对象作为键
  • WeakMap的键不计入垃圾回收
  • 目的在于，有时我们会想在某个对象上存一些数据，对这个对象进行说明，但是如果这个对象不需要了，我们需要手动删除这个引用，否则垃圾回收机制就不能释放空间。这个问题可以用WeakMap来解决，不用考虑WeakMap中的引用。
  • 只有get、set、has、delete

flex属性理解

Posted on 2020-08-06

flex属性代表什么

flex属性是三个属性的简写，分别是flex-grow、flex-shrink和flex-basis

flex-grow是指在还有父节点空余的空间的时候，子节点是否占用空余空间，占用的规则。
这规则是分配剩余空间的规则，如果都是1，则剩余的空间平均分配给各个子节点，注意是剩余空间，所以宽度就是原本的宽度加上剩余空间分配给自己的宽度。
默认0，即不放大

flex-shrink是指在父节点空间不足的时候，子节点的压缩情况。
如果需要减少30px，三个节点，大家的参数都是1，则每个减少10px。如果需要减少40px，参数是一个2两个1，则2的那个减少20px，1的减少10px。
默认1，即空间不足将项目缩小。若为0，则不缩小

flex-basis是指在分配空间之前，每个子节点占用的空间，根据这个属性，可以确定父节点的空间是有剩余还是不足。
默认auto，即节点原有的大小

flex的各种写法

flex的语法是这样的：
flex: none | auto | [ <'flex-grow'> <'flex-shrink'>? || <'flex-basis'> ]
这里面有很多特殊符号，这些符号的含义在下面解释一下：
|: 单管道符。意思是符号前后的值都是可以出现的，但是只能出现一个
[]: 中括号。中括号的意思是里面的东西都能支持，上面的含义就是三种属性都支持
||: 双管道符。就是或的意思，表示前后的内容可以分开独立合法使用。这是张鑫旭的解释，我的理解就是可有可无
?: 问号。表示一个或零个，可有可无

那么根据上面的语法，我们可以把flex的值分成三种，一个值，两个值，三个值：

一个值

如果是数值，如flex: 1，那就表示的是flex-grow的值，其他的两个是默认值，即flex-shrink是1，flex-basis是0%。这里有个坑，可能大家都认为flex-basis的默认值是auto，但是其实试验的结果并不是，而是0%。个人理解，应该是在所有的值都没有设置，或者设置为initial和auto的时候，flex-basis的值才是auto吧。
如果是长度值，如flex: 100px，那表示的是flex-basis是100px，设置为百分数也是代表的设置flex-basis，其他两个值都是1，这点也跟上面理解的默认值并不一样，主要是flex-grow不一样

两个值

两个值的话，第一个肯定是flex-grow的值，第二个值要根据情况来确定是flex-shrink还是flex-basis。
如果第二个数是数值，比如flex: 1 2，则2代表的是flex-shrink，flex-basis的值是0%。
如果第二个数是长度，比如flex: 1 100px，则第二个数代表flex-basis，flex-shrink的值是1。
有一个很扯的情况，flex: 1 0，这时flex-shrink是0而不是1，flex-basis是0%。这点是需要注意的。

三个值

三个值就是分别表示那三个属性了，这里不再赘述。

initial

等价于flex: 0 1 auto

auto

等价于flex: 1 1 auto

none

等价于flex: 0 0 auto

结语

其实理解了三个属性分别代表什么意思就好了，其他的在开发过程中可以自己去调试

Debounce

防抖是什么？
防抖是指一个事件在频繁触发的时候，不允许该事件执行，而是在一定间隔之后才执行，如果该事件在这个间隔内又被触发了，那么重新计时
根据这个描述，我们可以简单的写出一个debounce函数

函数中需要保存this和arguments的值，来修复this指向和event对象的问题

根据lodash的源码分步实现

由上面的防抖的基本思路，我们来一步一步实现lodash的防抖

第一步-基础

最基础的部分，我们根据上面第一部分的debounce原理，将代码分成了四个部分，方便之后的拓展

invokeFunction

  • 这个函数的目的是触发传入的func，并在触发之前做一些操作

timeExpired

  • 这个函数是在定时器执行到期之后触发的函数

startTimer

  • 顾名思义，这个是开始，或者更新定时器的函数

debounced

  • 一个包装了所有其他函数的基础函数，这个函数会被返回

第二步-时间的优化

上面的第一步已经基本实现我们的需求了，下面进行一些优化。
每次我们重置定时器的时候，用的都是wait，但是有可能出现时间的偏差。比如最后一次触发的时候是第10秒，我们定义的等待时间是5秒，但是由于一些原因呢，重置定时器时已经到第11秒了，那么只能在第16秒才能触发函数。这跟我们的预期是不一样的。那么我们就需要对重置定时器时的等待时间进行优化。

  • 首先我们需要定义一个变量来缓存上次触发debounce时的时间
  • 然后我们取当前的时间和这个时间进行对比，然后再和等待时间wait对比
  • 这次我们修改一下timerId定义和处理的方式
  • 加入canInvoke函数，来判断应该重置timerId还是触发函数

第三步-加入trailing参数和trailingEdge函数

防抖有两种触发方式，一种是前置触发，一种是后置触发。后置触发就是正常的在高频操作之后几秒后进行触发。前置触发就是在我们第一次触发这个操作的时间就触发一次这个函数。我们可以加入两个参数来控制前置还是后置。前置为leading，后置为trailing。

  • 首先我们要定义一个trailing变量，并初始化为true，因为默认的都是后置。
  • 参数中加一个options，里面定义trailing，然后我们需要对其进行校验
  • 添加一个trailingEdge函数
  • 更改调用方式，不调用原来的invokeFunction，而是调用trailingEdge函数

第四步-加入leading参数和leadingEdge函数

leading是指前置触发，也就是说，一开始触发debounce就要触发

  • 首先定义一个leading变量，初始化为false
  • 在options里面加入leading，并赋值
  • 添加leadingEdge函数，并在最开始调用，leadingEdge函数中要判断是否要立即调用函数，并初始化定时器

第五步-加入maxWait

maxWait的意思是，经过maxWait时间后，事件如果还没触发，则必须触发一次
如何实现呢，lodash是在remainingWait上面做文章，如果有这个maxWait，则会看看距离最近的一次触发的时间，和maxWait哪个长，如果到了maxWait的时间了，则触发。

  • 首先需要加入一个lastInvokeTime变量，来记录最后一次调用的时间
  • 然后加入maxWait参数
  • 更改remainingWait函数和canInvoke函数，兼容这种情况
  • 修改debounced函数，在可以触发的时候，如果是因为maxWait的原因，重置定时器，触发函数

这样基本上就是把lodash的防抖函数都扒的差不多了，除了一些取消和立即执行的函数，和一些其他附加的东西，这些之后再补充，不过也不是特别重要

二分法

上面的笑话简单讲了二分法的基本用法
在有序的数组中，查找一个特定的数，可以用到二分法，简单来说就是，取中间值，然后比较，然后再重新划分区域，然后再取中间值，然后再比较
但是这个简单的算法，细节非常重要，查找区间是否完整，查找区间如何调整，最终怎么取值，都是细节
二分法总得来说有三种应用场景，一是查找值，二是查左边界，三是查右边界。下面会一一讲解。

查找值

BFS-广度优先搜索

什么是广度优先搜索

广度优先搜索，就是系统的搜索整个图或者树，直到找到目标节点为止。
从根节点或者起始节点开始，将根节点的所有子节点加入队列当中，然后遍历子节点并将子节点的子节点加入队列，简单的说就是一层一层的遍历，一圈一圈的遍历。

实现方式

  • 首先将根节点放入队列中。
  • 从队列中取出第一个节点，并检验它是否为目标。
    • 如果找到目标，则结束搜索并回传结果。
    • 否则将它所有尚未检验过的直接子节点加入队列中。
  • 若队列为空，表示整张图都检查过了——亦即图中没有欲搜索的目标。结束搜索并回传“找不到目标”。
